# Gaxera v0.9.0 Release Notes — Hardware Enablement & Driver Infrastructure

> **Release Date:** 2026-07-23

> **Initiative:** Initiative #5 — Hardware Enablement & Driver Infrastructure (`docs/roadmap/roadmap_v09.md`)

---

## Highlights & Overview

Gaxera v0.9.0 completes full hardware enablement, driver framework, VirtIO transport, isolated ring-3 storage and network driver servers, and SMP multi-core scheduling across Milestones 0.9.0 through 0.9.5.

---

## Key Achievements & Feature Summary

### 1. PCIe ECAM Bus Discovery & PCI Server (`ADR 0027`)
- Unprivileged ring-3 `pci_bus_server` enumerating PCIe ECAM space (Buses 0..255, Devices 0..31, Functions 0..7).
- BAR parsing, IO/MMIO classification, and capability delegation to driver processes.

### 2. Driver Framework & Contiguous DMA Infrastructure (`ADR 0028`)
- Capability-authorized DMA physical memory allocations (`ContiguousFrameHandle`) and BAR `Mapping` delegation.

### 3. VirtIO Foundation & Virtqueue Transport (`ADR 0029`)
- Modern VirtIO 1.0 transport (`libgaxera::virtio`), allocation-free Virtqueue descriptor table management, available/used rings, size negotiation, and MMIO notification address calculation.

### 4. VirtIO Block Storage Server (`ADR 0030`)
- Isolated ring-3 `virtio_block_server` formatting 3-part descriptor chains with single-owner `BlockRequest` RAII DMA cleanup.

### 5. SMP Multi-Core Scheduling & Per-CPU Runqueues (`ADR 0031`)
- ACPI MADT AP core discovery, LAPIC bringup into 64-bit Long Mode, cache-isolated `CpuLocal` structures (`#[repr(C, align(64))]`), per-CPU runqueues, and deferred preemption (`need_resched`).

### 6. VirtIO Network Server & Zero-Copy Packet IPC (`ADR 0032`)
- Isolated ring-3 `virtio_net_server` formatting 2-part descriptor chains with `VirtioNetHeader` (12 bytes), dual `RX_VQ`/`TX_VQ` queues, `PacketBuffer` RAII state machine, event-driven WaitSet IPC, and client crash buffer reclamation.
