## Executive Summary

Gaxera `v0.9.1-recovery` is the formal, machine-verified release of the v0.9 Hardware Enablement milestone. It resolves all architectural drift, build-graph omissions, unsound address-space cloning, loose slab memory management, and ABI incompleteness present in the initial draft tag `v0.9.0`.

This release introduces:
1. **ContiguousFrame Kernel Object (`ObjectType::ContiguousFrame = 14`):** Real, kernel-owned, capability-authorized 2ⁿ contiguous physical frame allocations for DMA without userspace physical address synthesis.
2. **Bounded Mapping Subregion Derivation:** Strict parent-bounded subregion window derivation with page alignment and W^X capability enforcement.
3. **5-Level Global Lock Hierarchy:** Documented total lock ordering contract (Level 0 through Level 4) preventing deadlocks across kernel subsystems.
4. **Hardened SlabCache & ABI Parsing:** Double-free, interior pointer, and zero-size type validation in `SlabCache<T>`, alongside safe `try_decode` methods for all IPC wire messages.
5. **64-CPU SMP State Isolation:** Cache-line aligned (64-byte) `CpuLocal` structures and isolated per-CPU state structures.
6. **W^X Process ELF Loader Enforcement:** Prevention of simultaneous Write + Execute program header mappings.
