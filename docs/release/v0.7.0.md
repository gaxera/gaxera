# Gaxera Release Notes — v0.7.0

> **Tag:** `v0.7.0`  
> **Date:** 2026-07-23  
> **Initiative:** Epoch 2 — Multi-Client IPC & Event Multiplexing  
> **Baseline:** Gaxera v0.6 (`v0.6.0`)  

---

## Release Overview

Gaxera v0.7.0 evolves the microkernel's IPC mechanism from 1:1 synchronous rendezvous into a production-grade $N:1$ multi-client architecture with event multiplexing (`WaitSet`) and priority inheritance.

---

## Key Achievements & Feature Summary

### 1. Multi-Client `Endpoint` Caller Queueing (`ADR 0021`)
* Replaced single-caller rendezvous with bounded FIFO caller queueing (`MAX_ENDPOINT_CALLERS = 32`).
* Added `cancel_caller()` for queued thread cancellation and clean teardown on server crash or endpoint closure.
* Verified 16 concurrent client calls under UEFI QEMU (`test-ipc-multiclient`).

### 2. First-Class `WaitSet` Kernel Object (`ADR 0021`)
* Introduced `ObjectType::WaitSet = 13` and ABI opcodes `CreateWaitSet` (10), `WaitSetControl` (11), and `WaitSetWait` (12).
* Enables threads to block atomically on $N$ event sources (endpoints, notifications, timers) without fast-path heap allocations.
* Verified atomic event multiplexing under UEFI QEMU (`test-ipc-waitset`).

### 3. IPC Priority Inheritance (`ADR 0021`)
* Added `base_priority` and `effective_priority` tracking to `Thread`.
* Dynamically boosts server thread priority to match high-priority callers during active IPC rendezvous and restores base priority on reply.
