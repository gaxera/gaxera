# Gaxera v0.6.0 Release Notes — Core Memory Foundation

> **Release Date:** 2026-07-23  
> **Tag:** `v0.6.0`  
> **Initiative:** Initiative #1 — Memory Architecture (`docs/architecture/memory.md`)  

---

## Highlights & Overview

Gaxera v0.6.0 introduces a production-grade, non-leaking memory infrastructure and officially adopts the **Gaxera Engineering Workflow** for all post-v0.5 architectural initiatives.

Where v0.5 established capability isolation, preemptive scheduling, and microkernel IPC, v0.6 ensures that memory allocation is fully dynamic, leak-free, capability-governed, and zero-copy ready across isolated user address spaces.

---

## What Was Achieved

1. **Physical Frame Recycling & Page-Table Teardown (`ADR 0018`)**
   - Memory objects deallocate physical payload frames back to the frame allocator upon destruction.
   - User address spaces recursively dismantle intermediate page tables (`PML4` through `PML1`), ensuring zero memory leaks during process teardown.

2. **Two-Tier Kernel Slab Arenas & Growable Heap (`ADR 0019`)**
   - Core kernel objects (`Thread`, `Endpoint`, `CapabilitySpace`, `MemoryObject`, `X86AddressSpace`) use typed `SlabCache<T>` pages with O(1) slot allocation and automatic frame recycling.
   - The kernel heap dynamically expands by allocating physical frames on demand instead of being constrained to a fixed 2 MiB region.

3. **Subregion Memory Mapping & Unmap Primitive (`ADR 0020`)**
   - Memory mapping supports offset/length window projections into memory objects.
   - Added `OperationCode::UnmapMemory` (`Opcode 9`) syscall with user address validation and hardware TLB invalidation (`invlpg`).

4. **Zero-Copy Shared Memory Verification**
   - Verified bidirectional zero-copy communication across separate address spaces over derived `MemoryObject` handles under UEFI QEMU.

## Verification & Empirical Proofs

All deliverables were verified against empirical QEMU hardware test profiles:
- `test-frame-recycling`: 100 address space teardowns (0 frame leaks).
- `test-slab-allocation`: 1,000 object allocations across slab caches (100% slab page recycling).
- `test-unmap-memory`: Subregion projection and unmapping (TLB invalidated).
- `test-shared-memory`: Bidirectional zero-copy memory channels verified.
