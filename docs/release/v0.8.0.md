# Gaxera v0.8.0 Release Notes — Capability Microkernel Release

> **Release Date:** 2026-07-23  
> **Tag:** `v0.8.0`  
> **Initiative:** Initiative #4 — Capability Microkernel Architecture (`docs/roadmap/roadmap_v08.md`)  
> **Baseline:** Gaxera v0.7 (`v0.7.0`)  

---

## Highlights & Overview

Gaxera v0.8.0 establishes full capability-based microkernel infrastructure, completing Milestones 0.8.0 through 0.8.5. This release brings generation-tracked capability object arenas, hardware interrupt delegation, physical MMIO page mapping, type-safe `libgaxera` userspace runtime library, service discovery registry, and assembly fast-path context switches.

---

## Key Achievements & Feature Summary

### 1. Object System Generation Tracking & Tree Revocation (`ADR 0021`)
- Generation-tracked `ObjectArena` preventing use-after-free and slot reuse races.
- Recursive capability tree revocation across task CSpaces, invalidating child capability descriptors upon parent revocation.

### 2. Interrupt Delegation & WaitSet Notification Binding (`ADR 0022`)
- Unprivileged hardware IRQ delegation, IOAPIC vector routing, and IRQ-to-Notification binding.
- Direct wake of blocked WaitSet loops on hardware interrupt events.

### 3. MMIO Mapping Capabilities & Physical Range Isolation (`ADR 0023`)
- `Mapping` object encapsulation for physical memory page ranges.
- Safe page-aligned MMIO mapping into unprivileged driver address spaces.

### 4. Type-Safe `libgaxera` Userspace Runtime Library (`ADR 0024`)
- Created `libgaxera` runtime library providing `OwnedHandle` non-`Copy`/`Clone` ownership invariants and transient borrowed handles.

### 5. Service Naming, Discovery & Registry (`ADR 0025`)
- Ring-3 `init` service registry with `ServiceName` validation and capability delegation protocol.

### 6. Assembly Syscall Fast-Paths & Microbenchmarking (`ADR 0026`)
- Assembly syscall trampolines (`sysretq` / `syscall`), direct context-switch rendezvous (`try_direct_switch`), 64-byte inline register transfers, and QEMU TSC microbenchmark suite.
